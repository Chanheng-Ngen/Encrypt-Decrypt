I. Caesar Cipher

The Caesar Cipher is one of the oldest and simplest encryption techniques. It’s a type of substitution cipher where each letter in the plaintext is shifted a fixed number of places down or up the alphabet.

1.1 Assign numbers to each letter

A = 0, B = 1, C = 2, D = 3, E = 4, F = 5, G = 6, H = 7, I = 8, J = 9, K = 10, L = 11, M = 12, N = 13, O = 14, P = 15, Q =16, R = 17, S = 18, T = 19, U = 20, V = 21, W = 22, X = 23, Y = 24, Z = 25

1.2 Convert the plaintext into numbers

In the Caesar Cipher, each letter of the alphabet is assigned a number starting from A = 0, B = 1, C = 2, and continuing up to Z = 25. So when you take the word "MATH", you convert each letter into its number based on this system:

  • M is the 13th letter of the alphabet, but since we start counting from 0, M = 12.
  • A is the 1st letter, so A = 0.
  • T is the 20th letter, so T = 19.
  • H is the 8th letter, so H = 7.

1.3 Encrypt the numbers by adding 3 to each

First, convert each letter to its number using the standard Caesar Cipher system, where A = 0, B = 1, ..., Z =25.

  • M becomes 12
  • A becomes 0
  • T becomes 19
  • H becomes 7

Next, apply a Caesar Cipher shift of +3 by adding 3 to each number:

  • 12 + 3 = 15
  • 0 + 3 = 3
  • 19 + 3 = 22
  • 7 + 3 = 10

So the encrypted numbers are: 15, 3, 22, 10

1.4 Finally, convert those numbers back to letters:

  • 15 = P
  • 3 = D
  • 22 = W
  • 10 = K

So, after encryption, "MATH" becomes "PDWK" using a Caesar Cipher with a shift of +3.

III. RSA Cryptography

is a secure encryption method used to protect sensitive information, especially on the internet. It’s a type of asymmetric cryptography, meaning it uses two keys: a public key to encrypt data and a private key to decrypt it.

Rivest–Shamir–Adleman (Full word )

Public key: Shared with anyone. Used to encrypt messages.

Private key: Kept secret. Used to decrypt messages.

Why is RSA Secure?

It’s extremely hard to break because it relies on the difficulty of factoring a very large number (n) into its two prime factors. This is called the factoring problem, and it's what makes RSA strong.

Real-Life Uses of RSA:

  • HTTPS (secure websites)
  • Email encryption
  • Digital signatures
  • Secure communication apps
